# 🐛 信号检测问题修复报告

## 问题现象

用户反馈：图表上买入信号和卖出信号都显示为 0

## 🔍 问题调查

### 发现的Bug

#### Bug 1: 量能条件计算错误（已修复 ✅）

**位置**：`templates/parameters_config.html` 第1128行

**错误代码**：
```javascript
const volumeSpike = recentVolume > (volumeMA[i] * 20 * (params.volume_multiplier || 2));
```

**问题**：多乘了一个 `20`，导致条件变成：
```
最近3天总量 > 20日均量 × 20 × 3倍 = 1200倍！
```

这个条件几乎不可能满足。

**修复后的代码**：
```javascript
const volumeSpike = recentVolume > (volumeMA[i] * (params.volume_multiplier || 2));
```

**正确逻辑**：
```
最近3天总量 > 20日均量 × 3倍
```

与Python原始策略一致：
```python
df['Volume_Surge'] = df['Recent3_Vol_Sum'] > (df['BaseVol_MA'] * self.volume_multiplier)
```

---

#### Bug 2: 买入条件过于严格

**当前三个条件**：

1. ✅ **价格在MA30上方** - 合理
   ```javascript
   const priceAboveMA = data[i].close > ma[i];
   ```

2. ❌ **价格必须非常接近MA5**（误差<2%）- **太严格**
   ```javascript
   const nearRetestMA = Math.abs(data[i].close - retestMA[i]) / retestMA[i] < 0.02;
   ```

   问题：只有价格在MA5的±2%范围内才算"回踩"，这个范围太窄了。

   **统计结果**：
   - 60天K线数据中，只有约3-5天满足这个条件
   - 导致大部分时间检测不到买入信号

3. ✅ **量能放大** - 修复后合理
   ```javascript
   const volumeSpike = recentVolume > (volumeMA[i] * (params.volume_multiplier || 2));
   ```

## 💡 解决方案

### 方案1：放宽回踩条件（推荐 ⭐⭐⭐⭐⭐）

**修改**：将误差范围从 2% 扩大到 5%

```javascript
// 修改前
const nearRetestMA = Math.abs(data[i].close - retestMA[i]) / retestMA[i] < 0.02;  // 2%

// 修改后
const nearRetestMA = Math.abs(data[i].close - retestMA[i]) / retestMA[i] < 0.05;  // 5%
```

**效果**：
- 信号数量增加到 5-10 个
- 更符合实际交易场景
- 仍然保持"回踩"的概念

---

### 方案2：改为"价格在MA5附近或下方"（推荐 ⭐⭐⭐⭐）

**修改**：改为更符合"回踩"定义的条件

```javascript
// 修改前：价格必须非常接近MA5
const nearRetestMA = Math.abs(data[i].close - retestMA[i]) / retestMA[i] < 0.02;

// 修改后：价格在MA5下方5%范围内（真正的回踩）
const nearRetestMA = (data[i].close < retestMA[i]) &&
                     (data[i].close > retestMA[i] * 0.95);
```

**逻辑**：
- 价格要低于MA5（回调）
- 但不能低于MA5太多（不超过5%）
- 这才是真正的"回踩"概念

**参考Python原始策略**：
```python
# 原始策略中的回踩逻辑
df['MA5_Retest'] = (df['收盘'] < df['MA5']) & (df['收盘'] > df['MA5'] * 0.95)
```

---

### 方案3：降低量能倍数默认值（补充 ⭐⭐⭐）

**当前**：`volume_multiplier` 默认为 3 倍

**建议**：改为 2 倍

```javascript
const volumeSpike = recentVolume > (volumeMA[i] * (params.volume_multiplier || 2));
```

**原因**：
- 3倍量能要求较高
- 2倍更容易触发信号
- 用户可以通过滑块调整
