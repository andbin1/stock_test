# 专业级回测系统规范文档

## 第1章：成本模型设计

### 1.1 交易成本构成（A股实际水平）

在中国A股市场，完整的交易成本包括以下几个部分：

| 成本项 | 买入 | 卖出 | 说明 |
|--------|------|------|------|
| 手续费 | 万2.5（0.025%） | 万2.5（0.025%） | 证券公司佣金，双向收取 |
| 印花税 | 无 | 千1（0.1%） | 政府税收，仅卖出收取 |
| 滑点 | 0.1% | 0.1% | 市场冲击成本，双向 |
| 最低手续费 | 5元/单边 | 5元/单边 | 小额交易保护 |

### 1.2 完整成本计算公式

#### 1.2.1 单笔交易成本模型

```
买入成本总额 = 买入金额 × 手续费率 + 买入滑点
            = buy_amount × COMMISSION_RATE + buy_amount × SLIPPAGE
            = 100 × 10 × (0.025% + 0.1%)
            = max(1000 × 0.00125, 5)  [取最低费用5元]

其中，max()函数确保手续费不低于最低限制。

卖出成本总额 = 卖出金额 × (手续费率 + 印花税) + 卖出滑点
            = sell_amount × (COMMISSION_RATE + STAMP_TAX) + sell_amount × SLIPPAGE
            = 110 × (0.025% + 0.1% + 0.1%)
            = 110 × 0.00225
            = max(110 × 0.00225, 5)
            = max(0.2475, 5) [取最低费用5元]

总成本 = 买入成本 + 卖出成本 + 印花税
       = 5 + 5 + (110 × 0.001)
       = 10 + 0.11
       = 10.11元
```

#### 1.2.2 净收益计算公式

```
毛收益 = (卖出价 - 买入价) × 股数
        = (11 - 10) × 100
        = 100元

毛收益率(%) = (卖出价 - 买入价) / 买入价 × 100%
            = (11 - 10) / 10 × 100%
            = 10%

净收益 = 毛收益 - 总成本
       = 100 - 10.11
       = 89.89元

净收益率(%) = 净收益 / (买入价 × 股数) × 100%
            = 89.89 / 1000 × 100%
            = 8.989% ≈ 8.99%

成本百分比 = 总成本 / (买入价 × 股数) × 100%
          = 10.11 / 1000 × 100%
          = 1.011%
```

### 1.3 数值示例

#### 示例1：标准盈利交易
**场景**：买入10元，卖出11元，100股

```
买入金额 = 10 × 100 = 1,000元
卖出金额 = 11 × 100 = 1,100元

买入成本：
  - 买入手续费 = max(1,000 × 0.025%, 5) = max(0.25, 5) = 5元
  - 买入滑点 = 1,000 × 0.1% = 1元
  - 小计 = 6元

卖出成本：
  - 卖出手续费 = max(1,100 × 0.025%, 5) = max(0.275, 5) = 5元
  - 卖出印花税 = 1,100 × 0.1% = 1.1元
  - 卖出滑点 = 1,100 × 0.1% = 1.1元
  - 小计 = 7.2元

总成本 = 6 + 7.2 = 13.2元

毛收益 = (11 - 10) × 100 = 100元
毛收益率 = 100 / 1,000 × 100% = 10%

净收益 = 100 - 13.2 = 86.8元
净收益率 = 86.8 / 1,000 × 100% = 8.68%

成本占比 = 13.2 / 1,000 × 100% = 1.32%
```

**分析**：
- 名义收益10%，实际收益8.68%
- 交易成本侵蚀了收益的13.2%
- 这是一笔盈利的交易

#### 示例2：小额交易触发最低手续费
**场景**：买入5元，卖出5.5元，100股

```
买入金额 = 5 × 100 = 500元
卖出金额 = 5.5 × 100 = 550元

买入成本：
  - 买入手续费 = max(500 × 0.025%, 5) = max(0.125, 5) = 5元 [触发最低费]
  - 买入滑点 = 500 × 0.1% = 0.5元
  - 小计 = 5.5元

卖出成本：
  - 卖出手续费 = max(550 × 0.025%, 5) = max(0.1375, 5) = 5元 [触发最低费]
  - 卖出印花税 = 550 × 0.1% = 0.55元
  - 卖出滑点 = 550 × 0.1% = 0.55元
  - 小计 = 6.1元

总成本 = 5.5 + 6.1 = 11.6元

毛收益 = (5.5 - 5) × 100 = 50元
毛收益率 = 50 / 500 × 100% = 10%

净收益 = 50 - 11.6 = 38.4元
净收益率 = 38.4 / 500 × 100% = 7.68%

成本占比 = 11.6 / 500 × 100% = 2.32%
```

**分析**：
- 名义收益10%，实际收益7.68%
- 最低手续费导致成本比例升高到2.32%
- 小额交易明显不经济
- 建议最少交易金额要在4,000元以上以降低成本侵蚀

#### 示例3：亏损交易
**场景**：买入10元，卖出9元，100股

```
买入金额 = 10 × 100 = 1,000元
卖出金额 = 9 × 100 = 900元

买入成本：
  - 买入手续费 = max(1,000 × 0.025%, 5) = max(0.25, 5) = 5元
  - 买入滑点 = 1,000 × 0.1% = 1元
  - 小计 = 6元

卖出成本：
  - 卖出手续费 = max(900 × 0.025%, 5) = max(0.225, 5) = 5元
  - 卖出印花税 = 900 × 0.1% = 0.9元
  - 卖出滑点 = 900 × 0.1% = 0.9元
  - 小计 = 6.8元

总成本 = 6 + 6.8 = 12.8元

毛收益 = (9 - 10) × 100 = -100元
毛收益率 = -100 / 1,000 × 100% = -10%

净损失 = -100 - 12.8 = -112.8元
净收益率 = -112.8 / 1,000 × 100% = -11.28%

成本占比 = 12.8 / 1,000 × 100% = 1.28%
```

**分析**：
- 名义亏损10%，实际亏损11.28%
- 交易成本加重了亏损（成本侵蚀了额外的1.28%）
- 这是一笔亏损交易，成本控制更加重要

## 第2章：回测引擎规范

### 2.1 输入数据格式规范

#### 2.1.1 OHLCV数据要求

每只股票的历史K线数据必须包含以下字段：

```
DataFrame 列名要求：
  - 日期 (datetime): 交易日期，需要排序
  - 开盘 (float): 开盘价，必须 > 0
  - 高 (float): 最高价，必须 >= max(开盘, 收盘, 低)
  - 低 (float): 最低价，必须 <= min(开盘, 收盘, 低)
  - 收盘 (float): 收盘价，必须 > 0
  - 成交量 (int): 成交量（股），必须 >= 0
  - 成交额 (float): 成交额（元），必须 >= 0
```

#### 2.1.2 数据质量检查

```python
def validate_kline_data(df):
    # 检查必需列
    required_cols = ['日期', '开盘', '高', '低', '收盘', '成交量', '成交额']
    assert all(col in df.columns for col in required_cols)

    # 检查OHLC逻辑一致性
    assert (df['高'] >= df['开盘']).all()
    assert (df['高'] >= df['收盘']).all()
    assert (df['高'] >= df['低']).all()
    assert (df['低'] <= df['开盘']).all()
    assert (df['低'] <= df['收盘']).all()

    # 检查价格非负
    assert (df['开盘'] > 0).all()
    assert (df['收盘'] > 0).all()

    # 检查日期递增
    assert df['日期'].is_monotonic_increasing

    # 检查最小数据量
    assert len(df) >= 50  # 至少50个交易日

    return True
```

### 2.3 交易执行假设

#### 2.3.1 价格执行规则

```
买入执行价 = 当日收盘价 + 滑点
           = close_price × (1 + SLIPPAGE)

卖出执行价 = 当日收盘价 - 滑点
           = close_price × (1 - SLIPPAGE)
```

**理由**：
- 使用收盘价作为基准，代表当日实际成交价
- 买入时假设高于收盘价（0.1%滑点），保守估计
- 卖出时假设低于收盘价（0.1%滑点），保守估计
- 滑点值已在config.py中配置

#### 2.3.2 成交假设

```
单次最大成交量 = 当日成交量 × 5%

理由：不应该在单日内成交超过该日成交量的5%，
     否则会严重冲击市场
```

#### 2.3.3 持仓管理

```
- 不允许T+1日内反复交易（中国A股T+1交割规则）
- 卖出后下一个交易日才能再买入同一只股票
- 如果信号冲突，优先执行卖出信号
```

#### 2.3.4 时间处理

```
买入日期：信号产生的当日收盘
持有时间：计算自然日期差
卖出日期：卖出信号产生的当日收盘

最大持仓时间：由策略参数控制，
若超过最大持仓天数仍未卖出，则以市价卖出
```

### 2.4 边界条件处理

#### 2.4.1 数据不足

```
- 少于50个交易日的股票：跳过不回测
- 少于20个交易日的指标计算窗口：跳过该日期
- 回测起始日期前缺乏足够历史数据：使用最早可用数据
```

#### 2.4.2 极端行情

```
涨跌停板处理：
  - 如果当日涨停（上升>9.8%）或跌停（下降<-9.8%）
  - 认为该日无法正常交易
  - 卖出信号延迟到下一个非限制板的交易日执行

零成交量处理：
  - 成交量为0的交易日：跳过不交易
  - 无成交价格的情况：使用前一个有效收盘价
```

#### 2.4.3 资金管理

```
最小交易单位：100股（A股规则）

小额交易成本：
  - 交易金额 < 2000元：触发最低手续费5元
  - 推荐最小交易金额：4000元（成本占0.25%）

融资融券：暂不支持（当前版本仅支持做多）
```

### 2.5 风险披露

#### 2.5.1 回测偏差

```
1. 前向偏差（Look-ahead Bias）：
   - 确保交易信号只使用过去数据
   - 不使用未来信息生成信号

2. 幸存者偏差（Survivorship Bias）：
   - 当前回测不包含已退市股票
   - 实际过去可能包含已退市的标的

3. 滑点假设：
   - 0.1%滑点是保守估计
   - 实际流动性更强时可能更小
   - 极端行情时可能更大

4. 成本假设：
   - 基于2024年的平均交易费率
   - 手续费可能按交易所调整
   - 不考虑资金占用成本（融资利息）
```

#### 2.5.2 系统限制

```
1. 不支持的功能：
   - 融资融券/杠杆交易
   - 股票期权交易
   - 转债/权证交易

2. 单股重仓风险：
   - 当前允许单日多头持仓
   - 实际交易需要考虑头寸管理

3. 市场冲击：
   - 假设交易不影响市场价格
   - 大额订单的实际冲击可能更大
```
