## 🚀 快速开始

### 1. 启动应用
```bash
cd "D:\ai_work\stock_test"
python app_with_cache.py
```

### 2. 打开浏览器
```
http://localhost:5000/
```

### 3. 切换到回测页面
- 点击顶部标签栏的 "🔄 回测" 标签
- 在页面顶部应该看到 "💰 交易配置" 面板

---

## ✨ 核心功能速查

### 面板位置
```
回测标签页 → 交易配置面板 (顶部)
```

### 包含的参数
| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| 初始资金 | 10k ~ 10M | 100,000 | 数字输入框 |
| 交易占比 | 1% ~ 99% | 20% | 下拉框(预设) + 自定义 |
| 手续费 | 0% ~ 100% | 0.1% | 百分比输入 |
| 滑点 | 0% ~ 5% | 0% | 百分比输入 |

### 预览功能
- 显示4个预览项: 初始资金、单笔交易额、手续费、滑点
- 修改任何参数后自动更新

## 🧪 快速测试场景

### 场景1: 默认配置
**步骤**:
1. 打开回测页面
2. 观察配置面板

**期望结果**:
- 初始资金: 100,000
- 交易占比: 1/5 (20%)
- 手续费: 0.1%
- 滑点: 0%
- 预览: ¥100,000, ¥20,000, 0.10%, 0%

### 场景2: 修改初始资金
**步骤**:
1. 清空初始资金输入框
2. 输入 500,000
3. 观察预览

**期望结果**:
- 预览中的初始资金: ¥500,000
- 单笔交易额: ¥100,000 (20% of 500k)

### 场景3: 选择预设交易占比
**步骤**:
1. 点击交易占比下拉框
2. 选择 "1/3 (33.3%)"
3. 观察预览

**期望结果**:
- 单笔交易额更新为 ¥33,300 (33.3% of 100k)

### 场景4: 自定义交易占比
**步骤**:
1. 点击交易占比下拉框
2. 选择 "自定义"
3. 输入 0.15
4. 观察预览

**期望结果**:
- 自定义输入框显示
- 单笔交易额: ¥15,000 (15% of 100k)

### 场景5: 保存配置
**步骤**:
1. 修改参数: 初始资金 200,000, 交易占比 1/10, 手续费 0.05, 滑点 0.5
2. 点击 "💾 保存配置"
3. 观察反馈

**期望结果**:
- 显示 "✓ 配置已保存" 提示
- 3秒后提示自动消失

### 场景6: 验证错误处理
**步骤**:
1. 输入无效的初始资金 (如 5000)
2. 点击保存

**期望结果**:
- 显示错误提示: "初始资金必须在 10,000 ~ 10,000,000 之间"

### 场景7: 运行回测
**步骤**:
1. 配置参数 (如: 初始资金 100k, 交易占比 20%, 手续费 0.1%, 滑点 0)
2. 保存配置
3. 输入股票代码 (如: 000001)
4. 点击 "运行回测"
5. 等待回测完成

**期望结果**:
- 回测结果中显示配置信息
- 配置数据应用到回测计算中

## 🆘 常见问题

### Q: 页面加载缓慢?
A: 检查后端是否正常运行: `python app_with_cache.py`

### Q: 配置无法保存?
A: 检查浏览器控制台是否有错误，查看Network标签中POST请求的响应

### Q: 预览不更新?
A: 刷新页面，检查JavaScript是否启用

### Q: 回测时参数未应用?
A: 确保已点击"保存配置"按钮，然后再运行回测
