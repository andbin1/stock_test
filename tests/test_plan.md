## 1. 风险指标模块测试（Risk Metrics Module）

### 模块职责
- 计算最大回撤（Maximum Drawdown）
- 计算夏普比率（Sharpe Ratio）
- 计算 Sortino 比率
- 计算胜率和盈亏比
- 处理边界情况

### 测试用例详情

#### TC-Risk-001: 正常情况下最大回撤计算
**目的**: 验证最大回撤的计算正确性
**输入**:
- 收益序列: [1.0, 2.0, 1.5, 3.0, 2.0, 2.5]
- 初始资本: 100000

**期望输出**:
- 最大回撤 = (1.5 - 2.0) / 2.0 = -25%
- 正确识别从高点2.0到低点1.5的回撤

**测试步骤**:
1. 创建收益序列
2. 调用drawdown计算函数
3. 验证返回值是否正确

**验收标准**:
- 回撤值准确到小数点后两位

**测试步骤**:

**验收标准**:
- 返回值为0，无异常

## 2. 交易成本计算模块测试（Cost Calculation Module）

### 模块职责
- 计算手续费（Commission）
- 计算印花税（Stamp Tax）
- 计算滑点成本（Slippage）
- 处理最低手续费规则
- 计算净收益

### 测试用例详情

#### TC-Cost-001: 标准交易成本计算
**目的**: 验证正常交易的成本计算
**输入**:
- 买入价: 10.0元
- 卖出价: 11.0元
- 股数: 100股
- 手续费率: 0.05%（万5）
- 印花税: 0.1%（千一）
- 滑点: 0.05%

**期望输出**:
```
买入手续费 = 10.0 * 100 * 0.05% = 5.0元
卖出手续费 = 11.0 * 100 * 0.05% = 5.5元
卖出印花税 = 11.0 * 100 * 0.1% = 11.0元
买入滑点 = 10.0 * 100 * 0.05% = 5.0元
卖出滑点 = 11.0 * 100 * 0.05% = 5.5元
总成本 = 32.0元
```

**测试步骤**:
1. 输入交易参数
2. 调用calculate_trading_cost()
3. 验证各项成本

**验收标准**:
- 精确到分（0.01元）
- 每项成本计算无误

---

#### TC-Cost-002: 小额交易最低手续费处理
**目的**: 验证最低手续费规则的应用
**输入**:
- 买入价: 2.5元
- 卖出价: 2.75元
- 股数: 100股（低成本交易）
- 最低手续费: 5元
- 手续费率: 0.05%

**期望输出**:
```
买入金额 = 2.5 * 100 = 250元
手续费按费率 = 250 * 0.05% = 0.125元 < 5元最低
实际买入手续费 = 5元（取最低值）
```

**测试步骤**:
1. 计算费率手续费
2. 与最低手续费比较
3. 取较大值
4. 验证结果

**验收标准**:
- 手续费正确应用最低值

---

#### TC-Cost-003: 亏损交易成本
**目的**: 验证亏损交易的成本计算正确性
**输入**:
- 买入价: 10.0元
- 卖出价: 9.0元
- 股数: 100股
- 成本率: 0.2%（双边）

**期望输出**:
```
毛亏损 = (9.0 - 10.0) * 100 = -100元 (-10%)
总成本 ≈ 32元
净亏损 = -100 - 32 = -132元 (-13.2%)
```

**测试步骤**:
1. 输入亏损交易
2. 计算毛亏损和总成本
3. 计算净亏损
4. 验证成本仍然被正确扣除

**验收标准**:
- 亏损交易的成本不被忽略
- 净亏损包含交易成本

---

#### TC-Cost-004: 极小交易（触发最低手续费的边界）
**目的**: 验证超小额交易的处理
**输入**:
- 买入价: 1.0元
- 卖出价: 1.01元
- 股数: 100股
- 交易金额: 100元（非常小）

**期望输出**:
```
毛收益 = (1.01 - 1.0) * 100 = 1元 (1%)
成本 = 最低手续费 * 2 ≈ 10元
净收益 = 1 - 10 = -9元（亏损）
```

**测试步骤**:
1. 输入超小交易
2. 验证最低手续费应用
3. 检查净收益为负

**验收标准**:
- 小交易因手续费导致亏损被正确计算

---

#### TC-Cost-005: 批量计算一致性
**目的**: 验证批量计算与逐笔计算的一致性
**输入**:
- 5笔独立交易
- 每笔交易参数不同

**期望输出**:
```
逐笔计算成本和 == 批量计算成本
总成本一致
```

**测试步骤**:
1. 分别进行5笔交易的成本计算
2. 求和
3. 调用批量计算函数
4. 验证两个结果相等

**验收标准**:
- 差异 <= 0.01元（浮点数精度）
