### 虚假交易的特征

所有虚假交易的共同特征：
1. ✓ 同一天买卖（买入日期 = 卖出日期）
2. ✓ 收益率固定为 -0.1% （这是扣除的手续费）
3. ✓ 出现在连续买入信号的位置
4. ✓ 没有任何实际的价格变动

---

## 💡 关键洞察

### 1. 虚假交易的本质
```
原始版本问题:
  df.loc[i + hold_days, 'Sell_Signal'] = True

当有连续的Buy_Signal时:
  i=33: Buy_Signal=True  → 设置i+1=34位置为Sell_Signal
  i=34: Buy_Signal=True  → 设置i+1=35位置为Sell_Signal（同一天！）
  i=35: Buy_Signal=True  → 设置i+1=36位置为Sell_Signal（同一天！）

结果: 同一天既是卖出，又是买入
     → 价格不变 → 收益 = -0.1%（就是手续费）
```

### 2. 修复的核心改进
```
修复版本用持仓计数器:

for i in range(len(df)):
    # 当前持仓则计数递增
    if buy_date is not None:
        hold_counter += 1

    # 只有在计数达到hold_days时才卖出
    if buy_date is not None and hold_counter >= hold_days:
        # 卖出，重置状态

    # 只能在无持仓时买入
    if row['Buy_Signal'] and buy_date is None:
        # 买入，初始化状态

结果: 状态机清晰，避免冲突，完全正确
```

### 3. 为什么收益率变化这么大

原始版本虚假交易带来的虚假收益：
```
hold_days=3:
  虚假交易数: 1笔
  虚假收益: 1 × (-0.1%) = -0.1% （看起来减少收益）

  但实际上:
    原报告: 5盈2亏，胜率71.4%
    修复后: 3盈5亏，胜率37.5%

    问题在于虚假交易的交易对象不同：
    - 虚假交易占据了一些"位置"
    - 导致真实交易样本改变
    - 从而改变了整体的收益计算
```
